## 1. Загальна структура вікна

Застосунок — одне SPA-вікно (Tauri). Не перезавантажується між екранами, права панель замінює вміст.

```
┌───────────────────────────────────────────────────────────────┐
│  Sidebar (220px, фіксований)  │  Основна панель (решта)       │
│                               │                               │
│  · назва застосунку           │  topbar (заголовок + дії)     │
│  · навігація по секціях       │  ─────────────────────────    │
│  · лічильники / бейджі        │  контент (прокручується)      │
│                               │                               │
└───────────────────────────────────────────────────────────────┘
```

Sidebar завжди видимий. Не колапсується. Активний пункт виділено лівою бордюрною лінією.

## 2. Sidebar

### Структура навігації

```
[назва застосунку]
[кількість сесій сьогодні — підзаголовок]

─── ЗАПИСИ ───
  Поточна сесія
  Всі сесії

─── ЛЮДИ ───
  Контакти
  Невизначені          [лічильник]

─── ІНСТРУМЕНТИ ───
  Пошук
  Налаштування
```

### Поведінка лічильника "Невизначені"

- Відображає кількість фрагментів у `unknown_queue` зі статусом `unresolved`.
- Оновлюється в реальному часі під час запису.
- Зникає якщо черга порожня.
- Червоний колір лічильника — завжди, незалежно від кількості.

### Вимоги

- FR-SB-01: Sidebar завжди відображає актуальний лічильник невизначених мовців.
- FR-SB-02: Перехід між екранами не перезавантажує застосунок і не скидає стан запису.
- FR-SB-03: Під час активного запису пункт "Поточна сесія" показує мигаючий індикатор запису.

## 3. Екран: Поточна сесія

Основний робочий екран під час запису. Доступний завжди — навіть якщо запис не активний (показує стан "готовий до запису").

### 3.1 Topbar

Стани topbar:

| Стан | Відображення |
|---|---|
| Очікування | Кнопка "Почати запис" |
| Запис | Мигаючий індикатор + таймер + кнопки "Пауза" / "Зупинити" |
| Пауза | Статичний індикатор паузи + таймер заморожений + кнопки "Продовжити" / "Зупинити" |

#### Різниця між "Пауза" і "Зупинити"

**Пауза:**
- Мікрофон замовкає, VAD зупиняється.
- Таймер заморожується.
- Сесія залишається активною в пам'яті.
- Усі моделі залишаються завантаженими в RAM.
- Відновлення через "Продовжити" — без жодної затримки.
- Використовується для коротких перерв (< 10–15 хвилин).

**Зупинити:**
- Сесія завершується і записується в БД.
- Pipeline скидається (`session_id` закривається).
- Таймер скидається до нуля.
- Якщо в налаштуваннях увімкнено "Вивантажувати моделі після зупинки" — моделі звільняють RAM.
- Наступний запис потребує ініціалізації провайдерів (затримка 3–10 с залежно від моделей і опції вивантаження).
- Після зупинки відкривається повний транскрипт щойно записаної сесії.

### 3.2 Transcript pane (основна колонка)

Потік утерансів у хронологічному порядку. Нові утеренси апендяться знизу після кожної паузи VAD. Автоскрол до останнього утеренсу, якщо користувач не прокручував вручну.

#### Структура одного утеренсу

```
[аватар]  [ім'я мовця]  [час від початку сесії]  [мовний тег, якщо не основна мова]
          [текст транскрипту]
```

Деталі полів:

- **Аватар**: кольоровий круг з ініціалами для відомих мовців. Для невизначеного — сірий круг зі штриховим контуром і `?`.
- **Ім'я мовця**: для невизначеного — "Невідомий" приглушеним кольором.
- **Час**: відмітка від початку сесії у форматі `MM:SS`.
- **Мовний тег**: відображається якщо detected-мова утеренсу відрізняється від найчастішої мови сесії. Варіанти: `EN`, `UK`. Не відображається якщо збігається.
- **Текст**: для невизначеного мовця — приглушений колір до вирішення. Технічні ідентифікатори (слова що ASR повертає з великої літери або в backtick-форматі) виводяться моноширинним шрифтом.
- **Поточний утеренс** (ще обробляється): замість тексту — три анімовані крапки.

#### Вимоги

- FR-TR-01: Перший утеренс з'являється в UI протягом ≤ 2 секунд після паузи VAD.
- FR-TR-02: Ідентифікація мовця може затримуватись після тексту. Текст рендериться з тимчасовою міткою `?`, потім оновлюється без перезавантаження рядка.
- FR-TR-03: Автоскрол не блокується якщо користувач прокрутив вгору — відновлюється при ручному поверненні вниз або після нової паузи.
- FR-TR-04: Утеренси невизначеного мовця не блокують відображення наступних утерансів.

### 3.3 Side pane (~240px, права колонка)

Три незалежних блоки:

#### Блок "Мовці в сесії"

Картки для кожного унікального мовця що говорив у поточній сесії:
- Аватар + ім'я.
- Загальний час мовлення (оновлюється в реальному часі).
- Горизонтальний бар пропорційно до частки часу відносно загального.

Порядок: за часом першої появи в сесії.

#### Блок "Невизначений мовець"

З'являється якщо в поточній сесії є хоча б один нерозпізнаний фрагмент.
- Кількість фрагментів і сумарний час.
- Кнопка "Ідентифікувати →" — переходить на екран Черги невизначених, відфільтрований по поточній сесії.

Зникає якщо всі невизначені мовці вирішені.

#### Блок "Мова"

Два числа: відсоток UK і EN утерансів у поточній сесії. Оновлюється після кожного нового утеренсу.

### 3.4 Bottombar

Анімована waveform — живий сигнал з мікрофону. Суто індикативна, не клікабельна. При паузі — статична або прихована.

## 4. Екран: Всі сесії

### Список сесій

Зворотній хронологічний порядок. Можливе групування по днях (опціонально).

#### Структура картки сесії

```
[заголовок сесії]                        [дата / час]  [тривалість]
[аватари мовців: до 4, далі +N]          [кількість утерансів]  [мови]
[перший рядок транскрипту — preview]
```

Заголовок: або введений вручну, або автогенерований з першого утеренсу (перші 6–8 слів).

#### Вимоги

- FR-SL-01: Клік на картку відкриває повний транскрипт сесії в правій панелі.
- FR-SL-02: Транскрипт у правій панелі підтримує пошук по тексту (Ctrl+F або вбудований searchbar зверху).
- FR-SL-03: У відкритому транскрипті можна клікнути на ім'я мовця — перехід до його профілю в Контактах.
- FR-SL-04: Сесію можна перейменувати подвійним кліком на заголовок.
- FR-SL-05: Сесію можна видалити через контекстне меню або кнопку — з підтвердженням.

## 5. Екран: Черга невизначених

### Topbar

Заголовок "Черга невизначених" + кількість фрагментів що потребують уваги.

### Список фрагментів

Хронологічний порядок (найновіші зверху).

#### Структура картки фрагмента

**Header картки:**
```
[аватар ?]  [автомітка: "Невідомий мовець A/B/…"]
            [назва сесії · дата і час]
                                        [тривалість · N фрагментів]
```

Автомітки (`A`, `B`, …) — локальні в межах сесії, не глобальні.

**Body картки:**
```
▌ [цитата — найдовший або найрепрезентативніший утеренс]

Хто це?  [пілюля: контакт 1]  [пілюля: контакт 2]  [пілюля: контакт 3]  [+ Новий контакт]  [Пропустити]
```

#### Логіка пілюль-кандидатів

- Показуються лише контакти з `sim_score` > 0.65 (нижній поріг "можливо").
- Сортування: за спаданням `sim_score`.
- Максимум 3 пілюлі.
- Якщо жоден контакт не перевищує 0.65 — пілюль немає, тільки кнопка "+ Новий контакт".
- У пілюлі: аватар + ім'я. Тулітп при ховері: `sim_score` у відсотках.

#### Дії

| Дія | Результат |
|---|---|
| Вибрати пілюлю (контакт) | Фрагмент прив'язується до контакту, `speaker_segments.contact_id` оновлюється, картка зникає з черги анімовано |
| "+ Новий контакт" | Відкривається inline-форма: ввести ім'я → зберегти → фрагмент прив'язується → картка зникає |
| "Пропустити" | Фрагмент переміщується в кінець черги (`unknown_queue` оновлюється), картка зникає анімовано |

Після вибору контакту: embedding фрагмента додається до `voice_profiles` цього контакту.

#### Вимоги

- FR-UQ-01: Вибір контакту або "Новий контакт" оновлює `unknown_queue.resolved_contact_id` і `resolved_at` без перезавантаження.
- FR-UQ-02: "Пропустити" не видаляє фрагмент — переміщує в кінець черги.
- FR-UQ-03: Лічильник у Sidebar оновлюється миттєво після кожної дії.
- FR-UQ-04: Черга фільтрується по сесії якщо перехід відбувся через кнопку "Ідентифікувати →" в Side pane.

## 6. Екран: Контакти

### Ліва колонка (розширений sidebar або вбудована панель)

- Поле пошуку по імені.
- Список контактів: аватар + ім'я + кількість сесій.
- Кнопка "+ Новий контакт" внизу списку.

### Профіль контакту (права панель)

#### Hero-блок

```
[аватар 48px]  [повне ім'я]
               [дата першої зустрічі]
               [статистики: сесії · сумарний час · профілі · мови]
```

#### Вкладки

`Voiceprint` / `Сесії` / `Нотатки`

---

#### Вкладка: Voiceprint

Чотири характеристики у вигляді горизонтальних барів:

| Характеристика | Джерело даних | Відображення |
|---|---|---|
| Висота тону | Медіана F0 по всіх утерансах (`librosa.yin` або `parselmouth`) | "низький / середній / високий" |
| Темп мови | `len(words) / duration_sec`, нормований по базі | "повільний / середній / швидкий" |
| Енергія | RMS (`librosa.feature.rms`), середнє | числове, нормоване |
| Впевненість профілю | Середнє `sim_score` по всіх підтверджених ідентифікаціях | 0.00–1.00, зеленим якщо > 0.80 |

Характеристики рахуються **асинхронно** після завершення сесії, не в реальному часі. Кешуються в `contacts` або окремій `contact_stats`.

Кнопка **"Оновити профіль"**: примусово перераховує embedding як середнє по всіх підтверджених `voice_profiles` контакту.

#### Вкладка: Сесії

Список сесій в яких брав участь контакт:
- Назва сесії + дата.
- Час мовлення контакта в цій сесії.
- Клік → перехід до транскрипту сесії з підсвіченими утерансами цього мовця.

#### Вкладка: Нотатки

Вільне текстове поле (підтримка markdown). Автозбереження при втраті фокусу або після 2 секунд без введення.

#### Вимоги

- FR-CT-01: Профіль контакту відображає актуальну кількість сесій і час у реальному часі після нових записів.
- FR-CT-02: Видалення контакту — через кнопку з підтвердженням. При видаленні: `speaker_segments.contact_id` → `NULL`, фрагменти повертаються в `unknown_queue`.
- FR-CT-03: Кнопка "Оновити профіль" доступна лише якщо є ≥ 2 підтверджених фрагменти.
- FR-CT-04: Злиття контактів (merge) — вибір двох контактів → підтвердження → всі сесії і профілі переприв'язуються до одного.

## 7. Екран: Пошук

### Рядок пошуку

Завжди у фокусі при відкритті екрану. Shortcut: `Cmd/Ctrl + F` з будь-якого екрану.

Підтримка:
- Звичайний текст (FTS5 MATCH).
- Фразовий пошук у лапках: `"kubernetes deployment"`.
- Булеві оператори: `AND`, `OR`, `NOT`.

### Фільтри (панель під рядком пошуку)

| Фільтр | Тип | Опис |
|---|---|---|
| Мовець | Multi-select | Вибір одного або кількох контактів |
| Мова | Чекбокси | UK / EN / будь-яка |
| Дата | Date-range | Від — До |
| Сесія | Dropdown | Пошук в межах конкретної сесії |

### Результати

Групування по сесіях. Кожен результат:

```
[назва сесії]  [дата]
[аватар]  [ім'я мовця]  [час]
[фрагмент тексту з підсвіченим збігом]
```

Клік на результат → перехід до сесії, прокрутка до відповідного утеренсу.

#### Вимоги

- FR-SR-01: Пошук запускається після 300 мс паузи введення (debounce), не при кожному символі.
- FR-SR-02: Порожній запит з активним фільтром по мовцю — показує всі утеренси цього мовця.
- FR-SR-03: Результати показують максимум 50 збігів; кнопка "Показати ще" підвантажує наступні 50.

## 8. Екран: Налаштування

Розділи: Провайдери / Поведінка пам'яті / Зберігання / Загальні.

### 8.1 Провайдери

#### ASR — розпізнавання мовлення

Картки-радіокнопки для вибору моделі. Поточні варіанти:

| Модель | Розмір | Швидкість | Якість |
|---|---|---|---|
| `whisper tiny` | ~39 МБ | ~32× realtime | низька WER, прийнятна |
| `whisper medium` | ~1.5 ГБ | ~8× realtime | хороша, рекомендована |
| `whisper large-v3` | ~3.1 ГБ | ~3× realtime | найкраща WER |

Архітектура: нові варіанти додаються без зміни UI — список моделей читається з конфігу або реєстру провайдера.

Рядок статусу під блоком:
- Показує яка модель зараз завантажена і скільки RAM займає.
- Кнопка "Вивантажити" / "Завантажити" — ручне управління без перезапуску.

**Логіка переключення моделі:**
- Вибір нової моделі в UI зберігає зміну в `config.json`.
- Реальне переключення відбувається `lazy` — при наступному "Почати запис".
- Або негайно при кліку "Завантажити" в рядку статусу.
- Під час завантаження: рядок статусу показує "Завантаження…", кнопка "Почати запис" заблокована.

#### Embedding — voiceprint

Аналогічні картки:

| Модель | Розмір | Характеристика |
|---|---|---|
| `ECAPA-TDNN` | ~85 МБ | швидкий, рекомендований |
| `WavLM Large` | ~1.3 ГБ | точніший, більше RAM |

Архітектура провайдера — pluggable, нові моделі додаються через реєстр без зміни UI.

#### Diarization

Аналогічний блок (PyAnnote / NeMo / вимкнено). Можна вимкнути якщо потрібен тільки ASR без розділення по мовцях.

### 8.2 Поведінка пам'яті

**Перемикач "Вивантажувати моделі після 'Зупинити'":**
- За замовчуванням: вимкнено.
- Якщо увімкнено: після натискання "Зупинити" всі завантажені моделі вивантажуються з RAM.
- Наступний запис: затримка 3–10 с на повторне завантаження (залежить від моделей і диску).
- Якщо вимкнено: моделі залишаються в RAM між сесіями, новий запис стартує миттєво.

**Слайдер порогу ідентифікації:**
- Діапазон: 0.60 — 0.95.
- За замовчуванням: 0.82.
- Пояснення під слайдером: "Нижче 0.75 — більше авто-ідентифікацій, але ростуть хибні збіги. Вище 0.88 — майже всі незнайомі голоси йдуть у чергу."

### 8.3 Зберігання

- Відображення шляху до файлу SQLite БД.
- Розмір БД.
- Кнопка "Відкрити в Finder / Explorer".
- Перемикач "Backup при кожному закритті" (копіює `.db` у `backups/` з timestamp).
- Кнопка "Вибрати інше місце" — переміщення БД.

### 8.4 Загальні

- Мова ASR за замовчуванням (`uk` / `en` / `auto`).

## 9. Голосові характеристики — логіка розрахунку

Характеристики на вкладці Voiceprint — не частина embedding-вектора, а окремо обчислені ознаки поверх аудіо. Рахуються **асинхронно** після завершення сесії.

### Висота тону (F0)

```python
import librosa
f0, voiced_flag, _ = librosa.pyin(audio, fmin=50, fmax=400)
median_f0 = float(np.nanmedian(f0[voiced_flag]))
```

Або через `parselmouth` (Praat binding) для точнішого результату. Медіана по всіх підтверджених утерансах контакту. Відображення: `< 130 Hz` → "низький", `130–200 Hz` → "середній", `> 200 Hz` → "високий".

### Темп мови

```python
words = len(transcript.split())
duration_sec = (ended_ms - started_ms) / 1000
rate = words / duration_sec  # слів на секунду
```

Нормується по всіх контактах у базі. `< 2.0 сл/с` → "повільний", `2.0–3.5` → "середній", `> 3.5` → "швидкий".

### Енергія

```python
rms = librosa.feature.rms(y=audio)
mean_energy = float(np.mean(rms))
```

Нормується по всіх сесіях. Відображається як відносне число 0–100.

### Впевненість профілю

```python
avg_confidence = db.query(
    "SELECT AVG(sim_score) FROM speaker_segments "
    "WHERE contact_id = ? AND status = 'identified'",
    [contact_id]
).scalar()
```

Відображається як `0.00–1.00`. Зеленим якщо > 0.80, жовтим якщо 0.65–0.80, сірим якщо < 0.65.

### Оновлення embedding при "Оновити профіль"

```python
embeddings = db.query(
    "SELECT embedding FROM voice_profiles WHERE contact_id = ?",
    [contact_id]
).fetchall()
vectors = [np.frombuffer(e, dtype=np.float32) for e in embeddings]
new_embedding = np.mean(vectors, axis=0)
new_embedding /= np.linalg.norm(new_embedding)  # L2 normalize
db.execute(
    "INSERT INTO voice_profiles (contact_id, embedding, quality_score, ...) VALUES (...)",
    [contact_id, new_embedding.tobytes(), avg_confidence, ...]
)
```

Мінімум 2 підтверджених профілі для оновлення.

## 10. Стани моделей і пам'ять

### Стани моделі

```
NOT_DOWNLOADED → DOWNLOADING → LOADED → UNLOADED → LOADED
                                          ↑
                               (при наступному записі або ручному завантаженні)
```

| Стан | Відображення в UI | Дії |
|---|---|---|
| `NOT_DOWNLOADED` | "Не завантажено · X МБ на диск" | Кнопка "Завантажити" |
| `DOWNLOADING` | Прогрес-бар + відсоток | Кнопка "Скасувати" |
| `LOADED` | "Завантажено · X МБ RAM" | Кнопка "Вивантажити" |
| `UNLOADED` | "Не в пам'яті" | Кнопка "Завантажити" |

### Lazy load при старті запису

При натисканні "Почати запис":
1. Перевірити стан всіх потрібних моделей.
2. Якщо якась `UNLOADED` або `NOT_DOWNLOADED` → показати overlay "Завантаження моделей… X МБ / Y МБ" з прогрес-баром.
3. Після завантаження всіх → запуск запису без додаткового підтвердження.

### Вимоги

- FR-ML-01: Завантаження моделі не блокує UI — виконується в окремому потоці з прогресом.
- FR-ML-02: Застосунок не завершується поки модель завантажується або вивантажується.
- FR-ML-03: При зміні моделі ASR під час активної паузи (не запису) — нова модель завантажується, старий буфер транскрибується старою і тільки тоді відбувається переключення.

## 11. UX-принципи

### Near-real-time понад усе

- Перший утеренс з'являється в UI протягом ≤ 2 с після паузи VAD.
- Ідентифікація мовця може затримуватись — текст рендериться з `?`, потім оновлюється.
- Оновлення відбуваються in-place без перезавантаження рядків.

### Черга не блокує

- Невизначені мовці не зупиняють запис.
- Червоний лічильник нагадує, але не вимагає негайної дії.
- Чергу можна розбирати після сесії.

### Без деструктивних дій без підтвердження

- Видалення сесії — підтвердження.
- Видалення контакту — підтвердження + пояснення що відбудеться з фрагментами.
- "Пропустити" в черзі — не видалення, переміщення в кінець.
- Немає операцій що не можна скасувати.

### Мова слідує мовцю, не сесії

- Тег `EN` / `UK` на рівні утеренсу.
- Статистика мов у Side pane рахується по утерансах, а не по сесіях.
- В одній сесії обидві мови можуть чергуватись без будь-яких обмежень.

### Зміна конфігурації без перезапуску

- Вибір моделі, зміна порогу, toggle вивантаження — все вступає в силу без перезапуску застосунку.
- Єдина операція що потребує нового старту запису — зміна моделі (lazy load).
